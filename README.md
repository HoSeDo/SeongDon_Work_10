# 📌 UE5 모듈 및 플러그인 수동 구축 프로젝트

본 프로젝트는 언리얼 엔진 5(UE5) 환경에서 에디터의 자동 생성 기능을 사용하지 않고, 디렉터리 구조와 프로젝트 명세서(`.uproject`, `.uplugin`, `Build.cs`)를 직접 편집하여 독립된 모듈과 플러그인을 수동으로 구축하고 연동한 프로젝트입니다.

---

## 🚀 1. 핵심 구현 기능 (Features)

### 🔹 [필수 과제] 독립 모듈 구축 및 액터 스폰
* **모듈명**: `Test`
* **구현 내용**: 메인 프로젝트와 분리된 독립 C++ 모듈 `Test`를 수동 생성하고, 내부에 `ATestActor`를 구현. 플레이어 캐릭터가 게임을 시작하면(`BeginPlay`) `Test` 모듈의 `ATestActor`를 월드에 동적으로 스폰하도록 모듈 간 의존성 설정 완료.

### 🔹 [도전 과제] 격리된 플러그인 내 UObject 데이터 연동
* **플러그인명**: `Temporary`
* **구현 내용**: 타 프로젝트에 재사용 가능한 형태의 독립형 플러그인을 수동 생성하고, 내부에 캐릭터 데이터 저장용 `UObject` 자식 클래스(`UTemporaryCharacterData`) 구축. 메인 플레이어 캐릭터에서 해당 데이터를 동적 로드하여 초기 세팅값(HP, DMG)을 화면에 출력 성공.

## 🛠️ 2. 트러블슈팅 기록 (Troubleshooting)

### ❌ Issue 1: 수동 모듈 주입 후 IDE 인덱싱 마비 및 핵심 헤더 인식 불가능
* **현상**: Rider에서 플러그인 폴더와 코드를 수동으로 구성했으나, `CoreMinimal.h` 등 엔진 핵심 경로를 찾지 못하고 전체 검색(`Shift` 두 번) 등 IDE 기능이 마비됨.
* **원인**: 라이더는 단순히 하드디스크 폴더를 실시간 스캔하지 않고, 언리얼 빌드 툴(UBT)이 생성한 중간 프로젝트 가상 파일(`.json`)을 기반으로 인덱싱함. 수동으로 파일을 주입하면서 이 동기화 연결고리가 깨진 것이 원인.
* **해결**: 외부 윈도우 탐색기에서 `.uproject` 파일 우클릭 후 **`Generate Visual Studio project files`**를 실행. UBT를 통해 프로젝트 의존성 트리를 하드웨어 레벨에서 강제 재구성하여 해결함.

### ❌ Issue 2: 빌드 구조 갱신 중 임시 캐시 오류로 인한 물리 폴더 유실
* **현상**: UBT 프로젝트 파일을 재생성하는 과정 중, 임시 캐시가 꼬이면서 디스크 상의 `Plugins` 물리 폴더가 통째로 삭제됨. 그러나 라이더 편집창에는 파일 탭과 코드가 유령 형태로 그대로 살아있음.
* **원인**: 라이더의 **가상 파일 시스템(VFS)과 RAM 메모리 버퍼 캐싱** 기술 덕분. 디스크 실물은 지워졌으나, IDE가 성능 최적화를 위해 메모리에 올려둔 소스코드 버퍼가 가상 탭으로 유지되었던 것임.
* **해결**: 라이더를 종료하면 캐시가 비워져 코드가 영구 유실되므로, 툴을 켠 상태에서 코드를 외부 텍스트 에디터에 긴급 대피시킴. 이후 디스크 구조를 재구축하고 코드를 안착시켜 소스코드 증발 위기를 극복함.
---

> **💡 팀원 협업 프로세스**: 팀원이 원격 저장소에서 `Pull`을 받으면 `Binaries`나 `.sln`이 없는 상태입니다. 이때 팀원이 `.uproject` 우클릭 후 `Generate project files`를 실행하면, 해당 로컬 PC 환경에 맞는 빌드 트리가 완벽하게 재생성되어 즉시 협업이 가능해집니다.

## 💡 3. 프로젝트 회고 및 확장 설계 기준
* **모듈화의 의의**: 단순 에셋 마이그레이션이나 C++ 파일 복사는 내부 종속성과 컴파일 캐시를 꼬이게 만들지만, 명세서(`Build.cs`) 중심의 모듈/플러그인 분리는 엔진이 권장하는 가장 안전하고 깨끗한 재사용 방식임을 체득함.
* **컴포넌트 중심의 설계 기준 수립**:
  * **플러그인 (`Plugins/`)**: 채팅창, 인벤토리, 퀘스트 등 장르를 불문하고 다음 차기작에 **단 한 줄의 코드 수정 없이 통째로 복사해서 쓸 기능**들을 격리하여 구축함.
  * **메인 모듈 (`Source/`)**: 플레이어 조작, 적 AI, 탄환 메커니즘 등 **게임 고유의 기획과 규칙에 얽혀 변동성이 큰 콘텐츠**들을 효율적으로 관리하도록 폴더 구조를 정돈함.
